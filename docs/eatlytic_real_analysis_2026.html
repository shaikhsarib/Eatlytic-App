07b RAG Part — Live Context & Retrieval Architecture

Eatlytic implements a real-time web-augmented RAG (Retrieval-Augmented Generation) pipeline to inject live safety advisories, brand news, and nutritional data into LLM prompts. Below is a deep structural analysis of how it works, its vulnerabilities, and how to harden it.

How it works: Instead of searching static vector databases, Eatlytic formulates a search query from OCR tokens, crawls DuckDuckGo for live snippets, caches the results in two tiers, and passes them as a context block to the LLM to guide its analysis.

2. Deep-Dive Code Audit & Vulnerability Report

1
High-risk silent failure (ddgs blockages)
In research_engine.py, the live search relies on the ddgs package. DuckDuckGo aggressively blocks automated scrapers, returning 403 or 429 errors. In scan_orchestrator.py, this call is wrapped in a silent try...except. If DDG blocks you, the RAG feature fails silently, injecting empty strings and making the LLM act as a generic model without live context.
2
Naive query formulation (Generic search terms)
The query builder splits the OCR text and grabs the first 4 alphabetic words of length >= 4: _search_query = f"nutrition facts label {' '.join(_relevant[:4])} {_brand_hint}".strip(). On real labels, the first 4 words are almost always generic labels like "Nutrition Facts Serving Size" or "Amount Per Serving". Your query becomes: "nutrition facts label nutrition facts serving size". This fetches useless SEO blog posts instead of product-specific safety reports.
3
Hardcoded brand list is too small
The brand search hint only checks for: ["cadbury", "nestle", "amul", "britannia", "unilever"]. Major Indian brands like Haldiram's, Bikano, Slurrp Farm, Tata, or Yoga Bar will fail to append their brand hint, producing weak search contexts.
4
Unstructured context injection (No schema validation)
The search result is raw text concatenated with \n---\n and injected straight into the LLM system prompt. If the search results contain conflicting data, consumer reviews, or garbage SEO text, the LLM is highly prone to hallucinating inaccurate risk factors or Nova scores.

3. Recommendations to Harden the RAG Moat

  • Transition to Structured Retrieval: Instead of raw web scrapers, compile a local vector store (e.g., using FAISS/Chroma or pgvector on Supabase) containing verified FDA/FSSAI regulatory rules, Codex Alimentarius, and the Open Food Facts India dump.
  • Revamp Query Formulation: Formulate queries based on *extracted ingredients* rather than raw layout text. If the label contains "maltitol" and "hydrogenated vegetable fat", query for clinical reports or FSSAI limits for those specific chemicals.
  • Enrich Brand Telemetry: Pull brand name from a local dictionary or structured regex match before querying the web, instead of checking a 5-brand hardcoded list.
  • Enforce L2 Caching: Ensure L2 persistent database caching (SQLite/Supabase table research_cache) is 100% reliable to minimize external API roundtrips and avoid scraper bans in production.
